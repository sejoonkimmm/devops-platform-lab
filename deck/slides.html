Task 01 · Triage

Three failures. One of them invisible.

  • SilentPrice update accepted, never saved
    Kafka consumer fails on the event, but catch(Exception) swallows it and commits the offset. Client got 202. Nothing surfaced.
  • LatencyQuote endpoint ~46s
    A nested pricing loop issues a real DB query per iteration, and prices the whole catalog to return one SKU. Query fan-out, not "many reads".
  • 500Segment flash throws
    No mapping for flash, hits a null with no validation. vip works, so it's segment-specific.

Task 02 · Deployment

Built for 2k concurrent, and for the release that goes wrong.

# values.yaml — sized for the peak, not the average
autoscaling:
  minReplicas: 3
  maxReplicas: 12
  targetCPUUtilizationPercentage: 65
strategy: RollingUpdate   # canary via Argo for prod
probes:
  readiness: /actuator/health/readiness
  liveness:  /actuator/health/liveness
  • Rollback: Argo to last-good revision, DB migrations kept reversible.
  • Silent failure: health checks stay green, so a canary + metric/contract check catches wrong data and auto-rolls-back.
  • In chart vs separate: app + HPA in the chart; secrets in External Secrets, DB and Kafka managed.
